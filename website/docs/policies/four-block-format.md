# 4-Block Format

모든 에이전트 출력은 **예외 없이** 이 형식을 따릅니다. Slave 결과든 Master 최종 판정이든 동일합니다.

---

## 형식

```markdown
## 결론
최종 답변 또는 액션.
의사결정자가 이 블록만 읽고도 행동할 수 있어야 합니다.

## 근거
결론에 도달한 이유, 분석 과정, 참고 데이터, 대안 비교.

## 리스크
부작용, 엣지케이스, 주의사항.
기술적 / 비즈니스 / 운영 리스크를 구분합니다.

## 실행안
다음 단계, 구체적인 실행 방법.
즉시 실행 항목, 후속 확인, 담당자 권장.
```

---

## 왜 4-Block인가?

| 이점 | 설명 |
|------|------|
| **일관성** | 누가 어떤 모드를 쓰든 같은 구조 |
| **의사결정 가속** | "결론"만 보면 됨 — 바쁜 리더에게 필수 |
| **감사 추적** | 포스트모템에 그대로 사용 가능 |
| **Master 종합 용이** | 구조화된 입력 = 구조화된 판정 |

---

## 모드별 4-Block 활용

### @verify — 교차검증 결과

```markdown
## 결론
IAM 정책에 2건의 고위험 이슈 발견. 수정 없이 적용 불가.

| # | 항목 | Gemini | Codex | 채택 |
|---|------|--------|-------|------|
| 1 | kms:* 와일드카드 | ⚠ 감지 | ✅ 상세 분석 | Codex |
| 2 | Condition 블록 부재 | ❌ 누락 | ✅ 감지 | Codex |

## 근거
- Gemini: 5초 만에 kms:* 감지했으나, 대안 제시 부족
- Codex: 구체적 대안(kms:Decrypt, kms:GenerateDataKey) 제시

## 리스크
- kms:* 유지 시 키 삭제까지 가능

## 실행안
kms:* → kms:Decrypt, kms:GenerateDataKey로 변경
```

### @mobilize — Go/No-Go 판정

```markdown
## 결론
Go — 옵션 A 즉시 실행 (롤백)

## 근거
- Kiro: 3가지 가능 원인 중 정확한 범위 설정
- Codex: git log + 코드 대조로 근본 원인 특정
- Gemini: staging은 이미 수정, prod만 누락 확인
- 3-Agent 결과 간 모순 없음

## 리스크
- 롤백 시 4시간 내 다른 변경도 함께 롤백됨

## 실행안
1. ECS 롤백: 이전 Task Definition 배포
2. 커넥션 모니터링: 450 → ~150 복귀 확인
3. 후속: N+1 쿼리 핫픽스 PR 생성
```

### @build — 스펙 충족률

```markdown
## 결론
Kiro 스펙 충족률 95%. 1건 보완 필요.

| 스펙 항목 | Codex 구현 | 판정 |
|----------|-----------|------|
| 3개 모듈 분리 | ✅ | Pass |
| state mv 스크립트 | ✅ | Pass |
| state backup 절차 | ⚠ 언급만 | 보완 필요 |

## 근거
Codex가 모든 핵심 기능을 구현했으나 backup 자동화 누락

## 리스크
state backup 없이 state mv 실패 시 리소스 orphan 발생

## 실행안
migrate-state.sh 첫 줄에 자동 backup 추가
```

---

## 안티패턴

| 안티패턴 | 문제 | 올바른 접근 |
|---------|------|----------|
| 결론 없이 분석만 나열 | 읽는 사람이 직접 판단해야 함 | 결론을 먼저, 명확하게 |
| 리스크 섹션 비어있음 | 모든 결정에는 리스크가 있음 | "낮음"이라도 명시 |
| 실행안이 모호 | "확인하세요" 수준 | 구체적 명령어, 담당자, 기한 |
| 4-Block을 무시 | 일관성 파괴 | **협상 불가** — 항상 4-Block |
