# Multi-Agent CLI 소개

> **혼자 해도 되지만, 함께하면 놓치지 않는다.**

Multi-Agent CLI는 **4개의 AI CLI 에이전트**를 오케스트레이션하여 DevOps, SRE, 개발 작업에서 **단독 에이전트의 한계를 넘는 결과**를 도출하는 시스템입니다.

---

## 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **80/20 법칙** | 단독 에이전트는 80%의 답을 줍니다. Multi-Agent는 나머지 20%를 채웁니다 — 그 20%가 장애와 보안 사고의 차이입니다. |
| **4-Block Format** | 모든 출력은 결론 / 근거 / 리스크 / 실행안 형식입니다. 예외 없습니다. |
| **자동 모드 감지** | 프롬프트만 입력하면 Master가 최적 모드를 자동 선택합니다. 모드를 외울 필요가 없습니다. |
| **읽기 전용 Slave** | Slave는 분석만 합니다. 실제 변경은 Master가 사용자 승인 후 실행합니다. |

## 왜 Multi-Agent인가?

### 단독 에이전트 vs Multi-Agent

| 상황 | 단독 에이전트 | Multi-Agent |
|------|------------|-------------|
| **IAM 정책 리뷰** | "문법은 맞습니다" | Gemini가 패턴 감지 + Codex가 PoC 작성 → **kms:* 와일드카드 발견** |
| **프로덕션 장애** | "RDS 확인하세요, Redis도요" | Kiro가 트리아지 → Codex가 근본 원인 특정 → Gemini가 영향도 분석 → **10분 만에 복구** |
| **Terraform 리팩토링** | "모듈을 분리했습니다" (state 깨짐) | Kiro가 분리 설계 → Codex가 state mv 스크립트 작성 → **안전한 마이그레이션** |
| **DB 마이그레이션** | "ALTER TABLE 하세요" | Kiro가 전략 설계 → Codex가 Pre-check → Gemini가 영향 분석 → **Go/No-Go 판정** |

### 단독 에이전트의 근본적 한계

1. **자기 검증의 한계** — 자기가 빠뜨린 것을 자기가 발견하기 어렵습니다
2. **속도 vs 정밀 트레이드오프** — 하나의 에이전트로는 둘 다 잡기 어렵습니다
3. **관점의 단일성** — 보안, 성능, 설계를 동시에 최적화하기 어렵습니다

Multi-Agent는 이 한계를 **역할 분리**와 **교차 검증**으로 구조적으로 해결합니다.

```
└──────────┬───────────┬───────────┬───────┘
           │           │           │
     ┌─────▼─────┐ ┌──▼──────┐ ┌─▼────────┐
     │  Gemini   │ │  Codex  │ │   Kiro   │
     │  (Speed)  │ │(Precis.)│ │  (Spec)  │

     │           │ │         │ │          │
     │ 대량 분석  │ │ 정밀 코드│ │ 설계/스펙 │
     │ 로그 요약  │ │ 보안 검토│ │ 아키텍처  │
     │ AWS API   │ │ 테스트   │ │ 태스크분해│
     └───────────┘ └─────────┘ └──────────┘
```

## 7가지 실행 모드 미리보기

| 모드 | 명칭 | 에이전트 | 한 줄 요약 |
|------|------|---------|----------|
| `@ask` | 단독처리 | Claude | 간단한 질문, Master 혼자 충분 |
| `@scan` | 속도우선 | Gemini | 빨리 파악해야 할 때 |
| `@craft` | 정밀분석 | Codex | 정확해야 할 때 |
| `@design` | 설계먼저 | Kiro | 설계부터 해야 할 때 |
| `@verify` | 교차검증 | Gemini + Codex | 두 번 확인해야 할 때 |
| `@mobilize` | 총동원 | Kiro → Codex → Gemini | 절대 실패하면 안 될 때 |
| `@build` | 스펙→구현 | Kiro → Codex | 설계하고 바로 구현까지 |

:::tip 모드를 외울 필요 없습니다
프롬프트만 입력하면 Master가 키워드를 분석하여 최적 모드를 자동 선택합니다. `@모드`로 수동 오버라이드도 가능합니다.
:::
