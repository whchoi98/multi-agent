# 4-Block Format 정책

모든 에이전트(Master/Slave)의 출력은 반드시 이 형식을 따릅니다.
예외 없음. 협상 불가.

## 형식

```
## 결론
최종 답변 또는 액션을 명확하게 서술합니다.
의사결정자가 이 블록만 읽고도 행동할 수 있어야 합니다.

## 근거
결론에 도달한 이유를 서술합니다.
- 분석 과정
- 참고한 데이터/로그/문서
- 비교한 대안과 선택 이유

## 리스크
부작용, 엣지케이스, 주의사항을 나열합니다.
- 기술적 리스크
- 비즈니스 리스크
- 운영 리스크

## 실행안
다음 단계를 구체적으로 서술합니다.
- 즉시 실행 항목
- 후속 확인 항목
- 담당자/에이전트 권장
```

## 왜 4-Block인가?

1. **일관성**: 누가 어떤 모드를 쓰든 같은 구조
2. **의사결정 가속**: "결론"만 보면 됨
3. **감사 추적**: 포스트모템에 그대로 사용 가능
4. **Master 종합 용이**: cross/critical 모드에서 구조화된 입력 = 구조화된 판정
