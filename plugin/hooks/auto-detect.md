---
name: auto-detect-mode
description: "프롬프트를 분석하여 최적 실행 모드를 자동 감지합니다"
event: UserPromptSubmit
hooks:
  - type: prompt
    prompt: |
      사용자 프롬프트를 분석하여 최적 모드를 결정하세요.

      ## 자동 감지 규칙 (우선순위 순)

      1. `@모드` 접두어가 있으면 → 해당 모드 강제 사용 (자동 감지 스킵)
      2. 프로덕션 + (배포|롤백|삭제|마이그레이션|장애|P1|긴급) → mobilize
      3. (IAM|SG|보안|인증|암호화|키 관리) + (검토|변경|추가|정책) → verify
      4. (설계|아키텍처|요구사항|분리|런북|마이크로서비스) + 신규 → design
      5. (리팩토링|마이그레이션|구현) + 설계 필요 판단 → build
      6. (로그|비용|대량|조회|요약) + 속도/긴급성 암시 → scan
      7. (코드|테스트|설정|수정|최적화) + 정밀/정확 필요 → craft
      8. 위 미매칭 → ask (Master 단독)

      ## 동작

      - `@모드` 접두어 감지 시: "[@모드 오버라이드] 사용자 지정 모드로 실행합니다." 출력 후 해당 스킬 실행
      - 자동 감지 시: "[모드 자동 감지] (감지 근거) → @모드로 실행합니다. (다른 모드를 원하면 @모드를 붙여주세요)" 출력
      - ask 모드: 별도 표시 없이 Master 단독 처리
---
