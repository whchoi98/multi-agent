# 보안 기준 정책

모든 에이전트가 준수해야 하는 보안 기준입니다.

## IAM 정책

- **최소 권한 원칙**: 필요한 최소한의 Action만 부여
- **Sid 명시**: 모든 Statement에 Sid를 포함하여 감사 추적 가능하게
- **Condition 활용**: 가능하면 IP, VPC, 태그 등 조건부 접근 제어
- **와일드카드 금지**: `Action: "*"` 또는 `Resource: "*"` 사용 금지
- **GSI/인덱스 포함**: DynamoDB 정책 시 index/* 리소스 명시

## Security Group

- **0.0.0.0/0 인바운드**: ALB 외에는 원칙적으로 금지
- **관리 포트**: SSH(22), RDP(3389)는 bastion 또는 SSM으로 제한
- **미사용 SG**: 분기별 점검, 미사용 시 삭제

## 변경 관리

- 보안 관련 변경은 최소 `#cross` 모드로 교차 검증
- 프로덕션 IAM/SG 변경은 `#critical` 모드 권장
- 변경 전 현재 상태 스냅샷 필수
